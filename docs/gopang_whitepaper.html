Executive Summary

§ 0. 한눈에 보는 고팡

고팡(Gopang)은 현실 세계의 AI 쌍둥이(AI Parallel World)입니다. 사용자가 말 한마디를 건네면 고팡의 AI 비서가 의도를 파악하고, 필요에 따라 전문 AI Agent를 호출하며, 모든 과정을 사용자 본인의 기기에 암호화 기록합니다. 외부 서버에는 인덱스만 보관되므로 대규모 해킹의 표적이 존재하지 않습니다.

Philosophy & Vision

§ 1. 철학과 비전 — "AI 평행 세계"

1-1. 세 가지 구조적 문제

문제 1

데이터 중앙화

모든 개인 정보가 거대 플랫폼에 집중. 해킹 한 번으로 수억 명의 데이터 유출. 사용자는 자신의 데이터에 대한 실질적 통제권을 갖지 못함.

문제 2

서비스 파편화

음식 주문, 택시, 병원, 세금이 각기 다른 앱. 이들 사이의 연결은 사용자 몫. AI 시대에도 사람이 여전히 중개자 역할.

문제 3

거버넌스 부재

플랫폼 규칙은 소수 주주와 경영진이 결정. 사용자는 약관 변경을 통보받을 뿐, 결정에 참여하지 못함.

1-2. 고팡의 답

해결 1

데이터 주권 복원

모든 데이터는 사용자 본인의 기기(PDV)에 저장. 서버에는 해시 인덱스만 존재.

해결 2

AI 통합 포털

말 한마디로 50개 전문 Agent 자동 연결. AI 비서가 의도를 파악하고 최적 서비스를 호출.

해결 3

민주적 자치

주주도 임직원도 없음. 운영 비용은 수익자 부담, 시스템 변경은 DAWN 투표로 결정.

─────────────────                ─────────────

  └─ 결과 대기                  작업 처리
                                gwp.done({ summary, pdvData })
_onGwpMessage(e)  ←──────────── window.opener.postMessage

§ 5. 데이터 주권 — PDV

5-1. 로컬 저장 원칙

PDV(Private Data Vault)는 사용자 기기에 암호화 저장되는 개인 데이터 금고입니다. 고팡 서버에는 OpenHash 해시 인덱스와 PDV 6하원칙 요약(Supabase 백업용)만 존재합니다.

기존 구조:
  사용자 100만 명 → 중앙 서버 (1개) → 해킹 1회 = 100만 명 피해

고팡 구조:
  사용자 100만 명 → 각자 기기 (100만 개) → 해킹 1회 = 1명 피해
                         │
                    OpenHash 해시 인덱스만 공유 네트워크에 존재

§ 6. 위변조 불가 원장 — OpenHash

6-1. 5계층 구조와 PLSM

OpenHash는 SHA-256 기반 확률론적 계층 분산원장입니다. 거래마다 확률적으로 하나의 노드를 선택하여 에너지를 절감합니다.

계층단위선택 확률선택 범위
L1읍면동 (~7,000 노드)60%해시 mod 1000 → 0~599
L2시군구 (226 노드)20%600~799
L3광역 (17 노드)10%800~899
L4국가 (최소 13 기관)6%900~959
L5글로벌 (19+ 주체)4%960~999

6-2. 핵심 모듈

BIVM

잔액 불변성 검증

Σδ=0 원칙. 출금 합계 = 입금 합계. 위반 거래는 네트워크에서 자동 거부.

ILMV

양방향 계층 검증

L5→L1 하향 감사 + L1→L5 상향 모니터링 + 이웃 노드 교차 검증 동시 수행.

LPBFT

합의 알고리즘

L1 4노드 기준 ≤0.759ms. 5개 비상 조건 + 4개 복귀 메커니즘.

3요소 자기완결

① PDV 암호화 기록 ② OpenHash 앵커 ref ③ 삼중 서명 → 법원 증거 능력.

§ 7. 인증 시스템 — 인증없는 인증

고팡의 인증 시스템은 "로그인 없이 인증"합니다. 기기 자체가 신원입니다.

L0 — 기기 자동 인식

IPv6 + 기기 지문

접속 즉시 자동 완료. 로그인 버튼 없음. 일반 조회·AI 상담·시장 이용.

L1 — 얼굴 인증

Face ID (0.3초)

신고 접수·법률 자문·고액 결제. 스마트폰 카메라로 즉시 완료.

L2 — 지문 (WebAuthn)

FIDO2 / ECDSA

진료 기록·금융 거래. 서버에 지문 저장 없음. 개인키는 기기 보안 칩에만.

L3 — 4단어 시드

"정의의 물결 넘치는 꿈"

계정 복원·최고 보안. 가능 조합 수 50,000⁴ = 6.25×10¹⁸. 어디에도 저장 없음.

SSO 한 줄 구현

모든 하위 시스템은 </body> 직전에 단 한 줄만 추가합니다.

<script type="module" src="https://gopang.net/auth/subsystem-auth.js"></script>

window._onGopangAuth = async function(user) {
  // user.ipv6  : 사용자 GUID
  // user.level : 인증 레벨 (L0~L3)
  // user.via   : 인증 경로 (session/iframe/gwp)
};

§ 8. 법적 감시 — K-Law 상시 모니터

K-Law는 두 가지 모드로 동작합니다. 백그라운드 모니터는 모든 대화를 30초 쿨다운 간격으로 자동 분석하고, 판결 예측 서비스는 사용자 요청 시 K-Law v15.1을 실행합니다.

모든 고팡 대화 (30초 쿨다운)
    │
    ▼
K-Law 경량 모니터 (monitor_prompt.txt)
    ├── 안전 → 대화 계속
    └── 위험 감지
            ├── S1 경미 → 노란 경고 배너
            ├── S2 주의 → 법적 리스크 팝업
            └── S3 즉각 → K-Police / K-119 자동 호출

예방적 법률 시스템

PDV가 모든 통신과 거래를 6하원칙으로 기록하므로, 분쟁 발생 시 증거를 즉시 추출합니다. 전체 분쟁의 90%는 대화 단계에서 해결, 9%는 K-Law 조정, 1%만 실제 소송으로 이어집니다.

    ├── 교통사고  → K-119 + K-Police 동시 출동
    └── 재난(화재·홍수) → K-119 주도, K-Police 현장 통제

모든 사건은 PDV + OpenHash 앵커링으로 기록됩니다. ① PDV 암호화 기록 ② OpenHash 앵커 참조값 ③ 삼중 서명의 3요소가 갖춰지면 법원 증거 능력이 확보됩니다. 가해자가 PDV를 삭제해도 ②③이 블록체인에 앵커링되어 증거는 유효합니다.
